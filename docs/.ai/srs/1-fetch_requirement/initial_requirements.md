# HTTPS Server 模拟器 - 初始需求

## 项目概述
做一个HTTPS server端的模拟器，用于测试client端的报文请求。

## 核心功能点

### 1. Server端核心功能
- 能够提取HTTPS里面的报文内容
- 支持挂C接口回调做报文解析
- 将报文转译后返回给client端

### 2. Client模拟器
- 根据内容产生报文
- 和server建联
- 传出报文和接受报文
- 仅用于测试，不对外发布

### 3. 并发与性能要求
- 最多1个进程
- 进程内至多5线程
- 同时允许1000个client建联并行处理报文
- 需要考虑并行、高效、快速
- 需要考虑server端视角下的client生命周期问题

### 4. 平台支持
- Server端：Windows、Linux、Mac
- Client端：仅用于测试

### 5. Debug字段支持
- Client可能在HTTPS中加一个或多个debug字段（也可能没有）
- Server收到报文后，如果有对应字段，在报文解析正常后，按照报文内容做相应的调测处理

### 6. 非功能约束
- 无需考虑版本升级（卸载后重新安装即可）
- 无需考虑可靠性（掉电等）
- 无需考虑安全性问题（本身就是模拟器）

### 7. 技术栈
- 适配层（app界面、终端命令界面）：Python 3.8
- 其余部分：C++17
